## 1. 放大并美化首页按钮
在 [Hero.tsx](file:///f:/classos-new-main/components/Hero.tsx) 中，将目前仅显示图标且较小的“希沃”和“鸿合”下载按钮（`TransparentButton`）修改为带有文字说明的标准按钮（`Button`）。
- **具体改动**：
    - 将 `Apple` 图标按钮修改为 `下载 Seewo`。
    - 将 `Linux` 图标按钮修改为 `下载 HiteVision`。
    - 统一按钮样式，使其在视觉上与“下载 ClassOS”按钮保持一致，但在颜色上可能略有区分（例如使用半透明背景或边框以示次要地位，但尺寸相同）。

## 2. 移除下载代理功能
在 [Download.tsx](file:///f:/classos-new-main/components/Download.tsx) 中完全移除“使用代理下载”的功能。
- **具体改动**：
    - 删除 `useProxy` 状态变量。
    - 删除界面中的 `Switch` 开关及其说明文字。
    - 修改 `getDownloadUrl` 函数，使其直接返回原始的 GitHub 下载链接，不再拼接代理前缀。
    - 确保所有下载按钮直接连接到官方 GitHub Release。

## 3. 统一文档页面设计语言
重构 [Docs.tsx](file:///f:/classos-new-main/components/Docs.tsx) 的视觉设计，使其与首页（[index.astro](file:///f:/classos-new-main/pages/index.astro)）和下载页（[Download.tsx](file:///f:/classos-new-main/components/Download.tsx)）的设计风格完全一致。
- **具体改动**：
    - **背景与布局**：统一使用 `bg-mobai-background` 背景色和 `max-w-8xl` 容器。
    - **卡片设计**：将文档卡片的设计修改为与首页 `Block` 组件或下载页卡片相似的风格，包括一致的圆角、内边距、边框（`border-white/10`）和悬停效果。
    - **排版与配色**：使用相同的渐变色（`gradient-text`）和字体粗细。
    - **组件复用**：尽可能复用 `ui/Block.tsx` 中的设计模式，确保标题和内容的间距、大小与主页一致。

## 预期效果
- 首页下载入口更加清晰且统一。
- 下载过程更加直接，不再受代理服务稳定性影响。
- 用户从首页跳转到文档页时，视觉感受是连续且一致的。
